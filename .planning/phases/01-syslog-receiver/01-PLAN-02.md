## Plan 2: Web UI + Sender Feature + File Save

### Objective
Build the split-panel web UI with connection info, implement sender feature for testing, and add file save option.

### Tasks

#### T-06: Create index.html with split-panel layout
<read_first>
- None (new file)
</read_first>
<action>
Create assets/index.html with:
- Single HTML file with embedded CSS and JS
- Header bar showing listener status and port
- Two panels:
  - Upper panel (log display): Scrollable list showing last 20 logs, newest at top
    - Each log entry: timestamp, source IP:port, message
    - Auto-scrolls to newest, but user can scroll up to see older
  - Lower panel (connection info):
    - Listener address (IP:port)
    - Total logs received counter
    - Last update timestamp
- WebSocket connection to /ws
- On receive: prepend to log list, trim to 20
- Sender form: Destination IP, port, message input, Send button
- Save to file button
- CSS: Clean, minimal, monospace font for logs, dark theme
</action>
<acceptance_criteria>
- assets/index.html exists
- Browser shows two panels at /
- Logs appear in upper panel when received via WebSocket
- Connection info shows in lower panel
- Sender form visible and functional
</acceptance_criteria>

#### T-07: Implement sender HTTP endpoint
<read_first>
- main.go (Plan 1)
</read_first>
<action>
Add to main.go:
- SendRequest struct with DestIP, DestPort, Message fields
- handleSend function: POST /api/send, dials UDP to destination, sends syslog message
- Route: mux.HandleFunc("/api/send", handleSend)
</action>
<acceptance_criteria>
- POST /api/send accepts JSON with dest_ip, dest_port, message
- Sends UDP packet to destination
- Returns 200 OK on success, 500 on error
</acceptance_criteria>
